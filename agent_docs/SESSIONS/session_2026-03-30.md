**Root cause:** The `loading.tsx` files only work for server-side suspense boundaries, not client-side navigation. When clicking navbar links to admin/superadmin portals, Next.js does client-side navigation, but when `protectPage` runs on the server and redirects, the entire page (including navbar) unmounts before the redirect completes, causing a white screen with no spinner.

**Solution:** Created a global `NavigationLoadingProvider` that:
1. Detects when any `<a>` tag is clicked (including Next.js `<Link>` components)
2. Shows a fixed full-screen overlay with SpinnerLarge (z-index 9999)
3. Automatically hides when the pathname changes (navigation complete)

**How it works:**
- Provider wraps the entire app in root layout
- Listens for click events on all links
- Shows spinner immediately on click (before server processing)
- Hides spinner when Next.js pathname changes (after navigation completes or redirect happens)

**Tests:** Ready for testing - clicking admin/superadmin links from navbar should now show large spinner instead of white screen  
**Notes:** The spinner overlay has z-index 9999 to ensure it appears above all content including the navbar. This provides consistent loading feedback for all client-side navigation throughout the app.
